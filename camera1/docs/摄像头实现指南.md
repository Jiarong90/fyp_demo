# 摄像头与事故联动实现指南（新版）

本文档对应当前代码版本（`/ui2` 页面 + `camera1/server.js` + `camera1/py/compute_engine.py`），说明摄像头数据聚合、事故匹配与前端展示的完整流程。

## 1. 当前目标

1. 在地图上展示多来源摄像头点位（含实时图片与仅位置点位）。
2. 在 Dashboard / Alerts 展示实时事故，并关联附近摄像头图片证据。
3. 当事故附近无可用摄像头时，仍保留事故文本信息，不阻塞展示。

## 2. 当前架构

- 前端：`/Users/apple/Desktop/fyp_demo/UI 2/script.js`
- 后端：`/Users/apple/Desktop/fyp_demo/camera1/server.js`
- Python 计算引擎：`/Users/apple/Desktop/fyp_demo/camera1/py/compute_engine.py`

数据流：

1. 前端请求 `GET /api/cameras`，用于地图摄像头点位。
2. 前端请求 `GET /api/incidents`，用于 Dashboard/Alerts 事故列表。
3. 后端先拉取事故与实时摄像头，再调用 Python 执行“事故-摄像头最近匹配”。
4. Python 返回 `imageLink/cameraName/cameraDistanceMeters`，后端回传给前端展示。

## 3. 摄像头数据来源与接口

### 3.1 聚合接口

- `GET /api/cameras`
- 文件位置：`server.js` 中 `app.get('/api/cameras', ...)`

聚合来源：

1. data.gov.sg 交通摄像头（实时图片）
2. LTA Traffic Signal GeoJSON（信号点位）
3. SPF 红灯摄像头（KML）
4. OpenStreetMap camera 节点

返回结构（简化）：

```json
{
  "value": [
    {
      "CameraID": "...",
      "Latitude": 1.3,
      "Longitude": 103.8,
      "ImageLink": "https://...",
      "Name": "...",
      "Source": "...",
      "HasRealtimeImage": true
    }
  ],
  "meta": {
    "total": 1234,
    "realtimeWithImage": 90,
    "locationOnly": 1144,
    "warnings": []
  }
}
```

### 3.2 仅实时图片接口

- `GET /api/traffic-images`
- 返回 data.gov.sg 实时图片摄像头。

## 4. 事故与摄像头匹配（新版：Python）

### 4.1 事故接口

- `GET /api/incidents?source=live|mock&withImagesOnly=0|1&max=...`
- 文件位置：`server.js` 中 `app.get('/api/incidents', ...)`

### 4.2 匹配逻辑

后端调用：

- `attachNearestRealtimeCamera(...)`（异步）
- 内部优先走 Python：`runPythonCompute('enrich_incidents_with_cameras', payload)`
- Python 实现：`compute_engine.py` 的 `enrich_incidents_with_cameras`

匹配规则：

1. 对每个事故点计算到每个实时摄像头的 Haversine 距离。
2. 取最近摄像头；若最近距离 > 2000m，视为无可用摄像头。
3. 补充事故影响字段：`spreadRadiusKm`、`estimatedDurationMin/Max`。

容错回退：

- Python 失败时自动回退 Node 本地实现（`attachNearestRealtimeCameraLocal`）。

## 5. 前端展示位置

- Map View：展示摄像头点位与实时事故点（支持显示/隐藏按钮）。
- Dashboard：`Recent Updates` 与证据卡展示事故 + 摄像头图片。
- Alerts：事故卡片、View Details 中展示相关摄像头信息。

对应文件：`/Users/apple/Desktop/fyp_demo/UI 2/script.js`

## 6. 关键环境变量

在 `camera1/.env` 中配置：

```env
DATABASE_URL=postgresql://<user>:<pass>@localhost:5432/fyp_demo
PYTHON_BIN=python3
LTA_ACCOUNT_KEY=<可选，若使用 LTA 实时事故>
```

说明：

- `PYTHON_BIN` 默认是 `python3`，若本机命令不同可改。
- 未配置 `LTA_ACCOUNT_KEY` 时，事故接口会优先回退到 data.gov.sg 公开源。

## 8. 当前已知限制

1. 事故-摄像头匹配是“最近点”策略，不包含道路拓扑可达性。
2. 2000m 阈值是经验值，可按项目需要微调。
3. 实时图片有效性受上游源波动影响，可能临时失效。
4. 外部源（Overpass、公开数据）短时不可用时，会出现 warning 或回退数据。
